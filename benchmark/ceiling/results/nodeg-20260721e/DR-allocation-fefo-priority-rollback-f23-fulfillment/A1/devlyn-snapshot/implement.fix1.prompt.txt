## Root cause

In the `fulfillWave` function, `remaining` is built by sorting the `warehouses` array itself by `id`, then `flatMap`-ing each warehouse's OWN lots (sorted only by sku/expiry/lot within that single warehouse object). When two warehouse objects share an `id`, their lot lists never get merged and re-sorted together — each warehouse's lots stay grouped by source object, defeating the required global sku/expiry/lot tie-break within equal warehouse ids.

## Required fix

Flatten ALL `(warehouse, lot)` pairs into a single list first (each entry carrying its warehouse id), filter to `qty > 0`, THEN sort that single flat list once by warehouse id, then sku, then expiry, then lot id — instead of sorting warehouses and flat-mapping per-warehouse-sorted sublists. This makes the global ordering correct regardless of duplicate warehouse ids.
