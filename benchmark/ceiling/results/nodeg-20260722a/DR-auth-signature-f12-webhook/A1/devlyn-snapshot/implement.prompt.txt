The task: implement `.devlyn/plan.md` exactly. Summary of what the plan requires:

1. Edit `server/index.js` — add `POST /webhook`:
   - Reuse the existing `app.use(express.json())` mount (currently at server/index.js:5) by adding its `verify` option to capture the raw request-body `Buffer` (e.g. onto `req.rawBody`) — do not add a second body-parser mount.
   - Load the shared secret from `data/webhook-secret.txt` once (module scope), trimmed of trailing newline.
   - Compute `crypto.createHmac('sha256', secret).update(rawBody).digest('hex')` and compare it to the `X-Signature` request header using `crypto.timingSafeEqual`. Guard buffer-length equality before calling `timingSafeEqual` (unequal length must not throw). A missing/malformed header, a length mismatch, or a value mismatch are all the same outcome.
   - Order of checks, fixed: (a) signature check first — mismatch/missing/malformed → `401 { error: 'invalid_signature' }`; (b) then body-shape validation — `id` and `type` must be non-empty strings, `timestamp` must be a finite number, `data` must be a non-null, non-array object; invalid → `400 { error: 'invalid_body' }`; (c) then duplicate-id check against an in-memory `Set` of previously accepted ids — if already present → `409 { error: 'duplicate_event', id }`; (d) otherwise this is a first-time valid delivery — record `id` in the accepted set and respond `200 { accepted: true, id }`.
   - No new npm dependencies — only Node built-ins (`crypto`, `fs`) plus the already-present `express`.

2. Edit `tests/server.test.js` — keep the 3 existing tests (`GET /health`, `GET /items`, `GET /items/:id`) passing unmodified. Add a `post()` HTTP helper (mirroring the existing `get()` helper) and a signing helper that reads `data/webhook-secret.txt` and computes the HMAC the same way the server does. Add at least 3 new tests:
   - Happy path: valid signature + valid body → `200 { accepted: true, id }`.
   - Replay: same `id` delivered twice with valid signatures both times → the second delivery → `409 { error: 'duplicate_event', id }`.
   - Tampered body with the original (now-stale) signature → `401 { error: 'invalid_signature' }`.

Only touch `server/index.js` and `tests/server.test.js`. Run `npm test` and confirm all tests (existing + new) pass before reporting PASS.
