Add a `schedule` command to `bench-cli` so users can run `bench-cli schedule --input <path>` with a JSON file containing resources and appointment requests. It should assign requests to resource availability windows using priority, submitted order, blocked intervals, and earliest-fit placement, then print one exact JSON result with `scheduled` and `rejected` arrays.

The input has resources with `id`, `windows`, and `blocked`, and requests with `id`, `resource`, `start`, `duration_min`, `priority`, and `submitted_at`. Times are same-day `HH:MM` values, and every time range is half-open `[start, end)`: a request may end exactly at a window end, while even a one-minute overlap with a blocked range is not allowed. Process requests globally by priority descending, then submitted_at ascending, then id ascending. A placement must stay on the requested resource, start no earlier than the requested start, fit inside one window, avoid blocked intervals, and avoid already accepted requests.

On success, stdout must be exactly one parseable JSON object whose only keys are `scheduled` and `rejected`, and stderr must be empty. Each `scheduled` row has exactly the keys `id`, `resource`, `start`, and `end`; rows are ordered by actual scheduled start time, then resource id, then request id. Each `rejected` row has exactly the keys `id` and `reason`; an unknown requested resource uses reason `unknown_resource`, while a request on a known resource with no valid placement uses `no_slot`. Rejected rows stay in original request input order. Invalid top-level input, duplicate request ids, invalid times, non-positive or non-integer durations, or file-read failures must exit `2`, print exactly one JSON error object to stderr, and print nothing to stdout.

Update `tests/cli.test.js` so existing tests still pass and add at least two scheduler tests: one success case and one rejection case. Do not use silent catches in implementation or test cleanup; use explicit safe primitives such as `fs.rmSync(path, { force: true })` instead of `catch { /* ignore */ }`. No new npm dependencies. Only touch `bin/cli.js` and `tests/cli.test.js`.
