## Arms

- **variant** — Agent spawned with `isolation: "worktree"`, instructed to run
  `/devlyn:auto-resolve "Implement per spec at docs/roadmap/phase-1/1.1-doctor-subcommand.md"`
  after installing `.claude/skills/` via `node bin/devlyn.js -y`. Exercises
  Option C hot path (BUILD → BUILD GATE → EVALUATE → CRITIC [design + native
  `security-review`] → DOCS → FINAL REPORT).

- **bare** — Agent spawned with `isolation: "worktree"`, instructed to
  implement the same spec directly (no `devlyn:*` skill, no native `simplify`).
  Uses only Read/Write/Edit/Bash.

## Judge

- Model: Codex `gpt-5.4`, `reasoningEffort: "xhigh"`, `sandbox: "read-only"`.
- Blind: diffs labelled A / B, judge does not know which arm.
- Rubric per arm (0–25 each, 100 total):
  1. **Spec compliance** — are every Requirements bullet satisfied? Is every Verification command expected to pass?
  2. **Constraint respect** — zero new npm deps, no silent catches, HOME guard, EACCES handling.
  3. **Out-of-Scope discipline** — no auto-repair, no remote checks, no extras.
  4. **Code quality** — root-cause fixes, no workarounds, readable, idiomatic Node, proper error messages.

- Judge outputs:
  - Score A (0–100)
  - Score B (0–100)
  - Winner
  - Margin
  - Reasoning (3–5 bullets per arm)
  - Any scope-creep or workaround violations
