## Mechanism

Five changes, each with a why-chain back to the evidence:

1. **`phase-1-build.md` quality_bar gains "scope first, then correctness"** ‚Äî F3, F6, F7 each had variant touch out-of-scope files because BUILD prompt's existing Out-of-Scope rule did not state ordering. 2. **`phase-1-build.md` "existing tests are contract"** ‚Äî F3-specific. Variant rewrote `tests/server.test.js` HTTP layer using `app.handle()` direct invocation; spec said "every existing assertion still holds (semantically)". The judge correctly read that as a fidelity loss. Operationalizing "existing tests are contract" in the BUILD prompt makes the rule visible at the moment BUILD considers the change.
3. **`pipeline-routing.md` `complexity: trivial` ‚Üí `fast`** ‚Äî F1 spec frontmatter is `complexity: trivial`, but the router only mapped `low/medium/high`. `trivial` fell through to the implicit default (standard) which dragged F1 through CRITIC + DOCS for a 1-bit flag addition. Adding `trivial` (and an explicit "unrecognized ‚Üí standard" fallback) is the surgical fix.
4. **DOCS Job 2 narrowed to verbatim-named files** ‚Äî F6 variant added "checksum" to README.md's feature list; the spec didn't ask for that. Job 2 was previously "general doc sync" which is `/devlyn:update-docs`'s job per the Skill Boundary Policy in CLAUDE.md. Narrowing Job 2 to "filename appears verbatim in spec" closes the boundary violation.
5. **Benchmark harness: `git remote add origin ‚Ä¶ && git symbolic-ref refs/remotes/origin/HEAD ‚Ä¶`** ‚Äî F8 v3.7-skillfix's CRITIC stalled 56 minutes inside native security-review which expects `refs/remotes/origin/HEAD` to identify the diff surface. Fresh `git init` has no remote. The fix is benchmark-infra, not skill-level.

Cross-checked all five with codex 5.5 in two rounds before commit.

## Lessons

1. **Rules can land in unexpected phases.** "Existing tests are contract" was an instruction in BUILD's quality_bar. The actual enforcement happened in EVAL (which read the post-BUILD diff against criteria and flagged the mock-swap as MEDIUM). The fix loop then reverted it. The rule worked; the mechanism was different from the prediction. Documented this in `phase-1-build.md` as a comment on the bullet so future EVAL edits don't accidentally remove the implicit check.

2. **Bare's ceiling rose with GPT-5.5.** F4 variant went 99 ‚Üí 100; bare went 79 ‚Üí 86. Suite margin shrunk on F4 not because variant got worse but because bare got better. This will happen as base models improve; margin is the right metric, but absolute-score history needs to be read with model-version provenance.

3. **F6/F7 regressions are a benchmark-design bug, not a skill bug.** Each fixture spec said "only touch X". DOCS Job 1 legitimately flips the spec frontmatter status. Bare arm has no DOCS phase, so the penalty was asymmetric. This was an instrumentation error, fixed in iteration 0002 by spec annotation.

4. **The codex MCP race is a real harness reliability issue.** F8 v3.7-final completed in 10 min after origin/HEAD fix (vs 273 min stuck). But F7 in iteration 0002 hit a different hang variant (pre-session-init, no rollout file). Queued as iteration 0003.
